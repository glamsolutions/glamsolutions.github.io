ArcGIS Dashboards

Our team started this learning process with the use of ArcGIS Dashboards. It's a useful tool for generating a simple one page application that contains spatial information and allows for the inclusion of charts and numeric reporting.

Pros

Dashboards are a straightforward tool that does not require a lot of time to build a great application. The elements automatically adjust according to screen resolution.

Cons

It does not have any offline capabilities and does not support any additional spatial analysis outside of header filters. It allows for a lot of user interactivity, but the UX is not always intuitive.

ArcGIS StoryMaps

One of the ideas for our project was to include StoryMaps to act as a container for the final solution. This application allows users to share maps in the context of narrative text and other multimedia content. StoryMaps was used here to showcase the Web AppBuilder solution and provide context.

Pros

Great tool to incorporate maps, spatial data and media content together to display a narrative. It requires zero-coding skills and is simple to use for beginners

Cons

Many more design features are needed as currently only a few options are available. There are not enough formatting features and options when it comes to text and media elements (i.e., resizing, moving paragraphs/headers to different sections around the StoryMaps, etc.)

StoryMap Development

Once the required application has been completed, the developer can display the web solution along with providing context and other media to tell a story.

This is done by moving into the StoryMaps application and begins by importing the web solution (through the embed feature) and setting themes, headers, context and media through the text and image features.

Once customization of the solution is completed, the user can review and make any necessary edits before publishing.

App Builder Development

You begin by entering configure your web map preferably in ArcGIS Pro by adding all the necessary layers and data, customizing symbology and labelling and other map elements.

Once the web map in Pro is completed, the web map will be uploaded to ArcGIS Online through the use of Portal (in ArcGIS Pro).

Then you can customize web map (in map viewer) if necessary and once completed, log into ArcGIS Web App Builder and import web map to the platform.

Then in Web AppBuilder, configure the widgets, layout, data, web map and more to your liking, then review and preview application before launching.

ArcGIS WebApp Builder

Web AppBuilder is an exceptional way to visualize and work with data and allows user to configure map elements, widgets, GIS functionality, provide context and more. This platform allows configuration with tools already set so any user can customize their own workflows, visualize data, perform spatial analysis and more. This solution can easily display biomass data and users can easily compare between differing datasets.

Pros

It comes with a variety of widgets to provide advanced functionality and comes with great capabilities. There are various ways to provide context through infographics, graphs, texts and other media. It's also easy to use, no coding skills required and is flexible as it can be integrated with other platforms (i.e., StoryMaps and Experience Builder).

Cons

Many more design features are needed as currently only a few options are available. There are not enough formatting features and options when it comes to text and media elements (i.e., resizing, moving paragraphs/headers to different sections around the StoryMaps, etc.)

ArcGIS Experience Builder

ArcGIS Experience Builder allows you to quickly turn your data into interesting web apps without needing to write code. You are able to build apps and display them on a fixed or scrolling screen, on single or multiple pages.

Pros

Experience Builder is a great all in one type of tool that allows creators to make an app that can feel like an entire web page. It's also different from most ArcGIS app building products where the app you can create doesn't need to include a map.

Cons

The product is still very new and there isn't a lot of documentation to support development. The designing UX is not always clear and can be difficult to work with at times. The number of options for configuration can also feel overwhelming to new users.

Builder Development

Experience Builder has its own web page for creating products within ArcGIS Online, here you can start a new build from a large number of available templates. (Using a template would be recommended for new users)

From here, you are able to drag and drop and re-arrange widgets however you see fit. The left column is where you find and organize widgets and the right column is where you can configure the settings for charts, stastics, and other types of widgets.

Once all the settings are to preference, the product can then be published to the web.

Arcade Functionality

Arcade is a lightweight, portable expression language used to create custom content (like popups, etc.) in ArcGIS applications. It can manipulate text, perform mathematical calculations, and evaluate logical statements. Arcade expressions created in one application can be used in other applications. They can also be used in ArcGIS API for Javascript. Arcade has a rich library of functions to achieve practically any aim.

Pros

You can create dynamic displays for map/dashboard/application content can use same expression across multiple ArcGIS Applications.

Cons

Only works in ArcGIS (Pro, Online, Applications, API for Javascript)

VertiGIS Studio Web

VertiGIS Studio Web provides a foundation for customizing browser-based interactive mapping applications. Built to co-operate with Esri's JavaScript API, Web allows developers to extend the functionality of ArcGIS services with a more robust platform.

Pros

We did find the default "Web GIS" template option visually appealing once our map was in place and the docked side panel gives plenty of real estate for providing context and written directions to use the tools.

Cons

From prior encounters with VertiGIS/Geocortex web viewers as inexperienced users, we've found them tricky to operate as a first-timer. The tools are not the most intuitive to use and navigation between different pages of the docked pane takes some getting used to.

Development Findings

Since our targeted user is not likely to use this application frequently and may not be as spatially oriented as this development team, we concluded that the VertiGIS Studio Web platform is not optimal for our solution.

Conclusion

Throughout this process our team has experimented with a variety of web tools, each of them possessing great capabilities to push web GIS to the next level. However we ultimately chose ArcGIS Dashboards as the ideal solution for our problem.

We chose the end product based on a number of reasons such as time taken to develop, number of available widgets and function options, and ease of use for the end user. We found that ArcGIS Dashboards was the most straightforward without compromising too much on functionality.
