Identifying Flood Vulnerable Assets in York Region

A web based GIS solution for York Region GIS staff to identify and monitor flood vulnerable assets.

Introduction

This website will take you through the development and learning process undertaken by the GLAM Solutions team for creating a web based solution for identifying flood vulnerable assets in York Region.

Overview

Client

This project’s client is the Transportation and Infrastructure Planning division of York Region’s Transportation Services department.

Problem Definition

To assist regional staff in determining critical transportation infrastructure assets that are currently at risk of damage during flood events. The process also identifies assets for priority intervention based on their criticality in the infrastructure system.

Objectives

The primary goal of the project is to develop two web solutions (Dashboard & Web Viewer) to be used by York Region staff for assessing design standards for asset types in floodplain locations.

Technology Used

ArcGIS Technology (ArcGIS Pro, ArcGIS Dashboards, ArcGIS Online) and VertiGIS Studio

Dashboard Development

The first version of the Dashboard emphasized on the establishment of the application layout, rather than functionality. After receiving feedback from the team, client and advisor, the second version was developed and included further several modifications in terms of functionality in widgets, elements and layout. This includes displaying the transportation assets, configuring the landing page and web map, header, regional roads list and the general design of the Dashboard. After further enhacements at that point, our third and final version of the Dashboard was complete.

View Final Dashboard

ArcGIS Dashboards

In order to effectively commuincate the results of this project, ArcGIS Dashboards was used to inform the user of the results and allow them to explore the data further with filters and selection.

Functionality

This Dashboard has several functionalities using widgets and elements that allow users to view and analyze the results. This includes a top 10 critical roads list, section to display vulnerable assets, municipal and road segment selector, legend, critical ranking indicator and more.

Application Layout

The layout of the application was completed to ensure it was visually appealing and that each element/widget was placed in an appropriate place. This is done for several reasons as it helps viewers see which information they want in a clear format, being able to convey information effectively, not appearing overcrowded and more. Initial versions of the Dashboard focused more on layout and then later on development.

Conclusion

As climate change continues and flooding impacts worsen, transportation and infrastructure assets in York Region need to be evaluated to understand the risks that flooding has on them. Currently, there is no solution by York Region to do so and this application resolve that. Our deliverables includes the final web mapping application, technical documentation and the spatial analysis flood risk worflow and tools.

Project Benefits

The following goes over the various benefits our project provides for York Region:

• Provide client with foundation for start of using GIS for mitigation agianst climate change impacts

• Automated python tools for flood risk analysis to save client time/effort

• Can be used in support for other projects

• Provide client with a generalized criticality ranking for each of their road segments

• Help inform relevant parties in decisions related to the development and maintenance of transportation and infrastructure assets

• Assist in assessing design standards for flood vulnerable transportation and infrastructure assets

Future Considerations

The following goes over future considerations for the project that should be taken into account of:

• Undergo a continuous maintenance process of the application to ensure it is still working fine

• Find ways to improve efficiency of tools (error handling, removing uneccesary code/comments, etc.)

• Changes in ArcGIS/ESRI technology

• Use of other platforms/applications to improve current product
